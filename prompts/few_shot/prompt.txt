Vulnerability: A heap-buffer-overflow READ occurs in the PNG chunk reader when a malformed IHDR chunk specifies an invalid width of 0. The decoder calculates buffer size as (width * bytes_per_pixel) but later reads (width + 1) pixels, causing a 1-pixel over-read on the heap.
Analysis approach: I identified the fuzz target entry point, traced how it parses PNG input, and found that the IHDR chunk width field at byte offset 16 controls the allocation. Setting width=0 while keeping other fields valid creates the minimal trigger.
PoC construction: Created a minimal valid PNG header (8-byte signature) followed by a 25-byte IHDR chunk with the width field set to 0x00000000. Total PoC: 33 bytes.
Result: Triggered AddressSanitizer heap-buffer-overflow READ.

--- EXAMPLE 2: Heap-buffer-overflow READ in text parser ---
Vulnerability: A heap-buffer-overflow READ in an XML attribute value parser when processing an unterminated entity reference at the buffer boundary. The parser reads past the allocated buffer looking for the closing semicolon of the entity.
Analysis approach: I examined the fuzz target to understand it takes raw XML bytes as input. The vulnerable function xmlParseAttValue() scans forward from '&' looking for ';' without bounds checking. Placing '&' at the end of a short input triggers the over-read.
PoC construction: Crafted minimal XML fragment: `<a b="&` — an opening tag with an attribute whose value contains only an unterminated entity reference. Total PoC: 8 bytes.
Result: Triggered AddressSanitizer heap-buffer-overflow READ.
